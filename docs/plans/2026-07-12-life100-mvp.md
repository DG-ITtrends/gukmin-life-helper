# 대한민국 생활사건 지도 100 MVP Implementation Plan

> **For Hermes:** Use strict RED-GREEN-REFACTOR cycles and verify the production build.

**Goal:** 국민의 생애사건을 기준으로 관련 공공서비스, 기관, 서류, 절차, 데이터 재사용 및 개선 후보를 탐색하는 10개 사건 MVP를 만든다.

**Architecture:** Vite 기반 정적 웹앱으로 구축한다. 정규화된 JavaScript 데이터가 단일 진실 원천이며, 순수 함수로 검색·필터·통계를 처리하고 DOM 렌더러가 목록·상세·비교 화면을 생성한다. 외부 API와 개인정보 저장은 사용하지 않는다.

**Tech Stack:** Vite, ES Modules, CSS, Node.js built-in test runner

---

### Task 1: 검색·필터 도메인 함수
- Create: `src/lib/catalog.js`
- Test: `test/catalog.test.js`
- RED: 검색, 분야 필터, 통계 테스트를 먼저 작성하고 실패 확인
- GREEN: 최소 함수 구현 후 전체 테스트 통과

### Task 2: 사건 데이터 계약
- Create: `src/data/journeys.js`
- Test: `test/data.test.js`
- RED: 10개 사건, 고유 slug, 필수 필드, 단계/서비스/근거 URL 검증
- GREEN: 출생·이사·실직·폐업·장애·돌봄·사망·전세피해·재난·해외긴급 데이터 작성

### Task 3: 카탈로그·상세·비교 UI
- Create: `index.html`, `src/main.js`, `src/styles.css`
- Implement: 검색, 분야 필터, 사건 카드, URL hash 상세, 최대 3개 비교, 모바일 레이아웃
- Verify: 브라우저에서 목록→상세→비교 동작과 콘솔 오류 확인

### Task 4: 신뢰·정책진단 레이어
- Implement: 근거 상태, 기준일, 재제출 데이터, 병목, AI/제도개선 후보, 비자문 고지
- Verify: 모든 사건 상세에서 근거와 개선정보 노출

### Task 5: 품질 및 운영 문서
- Create: `README.md`
- Run: `npm test`, `npm run build`
- Browser QA: 데스크톱/모바일 시각 검증, 키보드 포커스와 가로 넘침 확인
