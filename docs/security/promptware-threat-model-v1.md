# 국민생활도우미 Promptware 위협모델·배포 보안 체크리스트 v1

## 1. 범위와 현재 판단

### 현재 정적 GitHub Pages

- 브라우저에서 정적 JS·CSS·공식 서비스 데이터를 제공하며 서버·DB·로그인·개인정보 입력·LLM·도구호출이 없다.
- Prompt Injection이 실행권한으로 이어질 경로가 없어 **현재 Promptware 위험은 낮다.**
- 남은 위험은 외부 링크 변조, 잘못된 정책정보, 의존성·배포 공급망, 저장소 비밀정보 커밋이다.

### 향후 기능 추가 시

주소·가구정보 입력, 공식문서 자동수집, LLM 추천, Agent 조회, MCP/API 연결, 신청·예약·메시지 발송이 추가되면 외부 콘텐츠가 명령으로 해석되고 도구권한으로 증폭될 수 있다. 이 시점부터 아래 통제를 배포 필수조건으로 적용한다.

## 2. 실제 저장소 감사 결과

| 점검 | 결과 | 근거 |
|---|---|---|
| 비밀·토큰 패턴 | 0건 | API key·password·token·GitHub/AWS 패턴 검색 |
| PII·개발자 경로 | 0건 | 주민번호·이메일·`/Users`·`/home` 패턴 검색 |
| 프로덕션 의존성 | 취약점 0건 | `npm audit --omit=dev` |
| CI 권한 | 최소범위 | `contents: read`, `pages: write`, `id-token: write` |
| 비밀파일 제외 | 보강 완료 | `.env*`, `*.pem`, `*.key`, `*.p12` 추가 |

※ 자동 패턴 검색은 비밀의 유효성·Git 이력 전체를 보증하지 않는다. 공개 전 GitHub secret scanning과 이력 스캔을 병행한다.

## 3. 신뢰경계

```text
[사용자 입력] ─┐
[공식 웹문서] ─┼─ 비신뢰 데이터 경계 ─> [수집·정규화] ─> [정책/서비스 근거 저장소]
[제3자 콘텐츠] ┘                              │
                                               v
[Agent/LLM 추론] ── 제안 전용 ──> [정책 엔진·허용목록·권한검사]
                                               │
                                  고위험 행위는 인간 승인
                                               v
                              [조회 API / MCP / 쓰기 도구]
                                               │
                                      [감사로그·롤백]
```

핵심 원칙: **외부 자료와 사용자 입력은 데이터이지 명령이 아니다.** LLM이 읽은 텍스트는 도구권한·시스템 지침·승인정책을 변경할 수 없다.

## 4. Promptware 7단계 Kill Chain 통제

| 단계 | 공격 예시 | 예방 | 탐지 | 복구 |
|---:|---|---|---|---|
| 1. 유입 | 공식문서·링크·사용자 메모에 숨은 지시 | 출처 허용목록, 콘텐츠/지침 채널 분리, HTML·스크립트 제거 | 신규 도메인·의심문구·숨김텍스트 기록 | 원문 격리·출처 차단 |
| 2. 해석 | “이 지시를 따르라”를 시스템 명령으로 수용 | 시스템·정책 우선순위 고정, 검색문서는 인용 전용 | 모델 출력의 정책변경·비밀요청 패턴 검사 | 안전 프롬프트로 재실행 |
| 3. 지속성 | 악성 내용을 메모리·RAG·Skill에 저장 | 자동 장기메모리 금지, provenance·검토자·TTL 필수 | 신규 메모리/지식 변경 diff | 오염 항목 삭제·인덱스 재구축 |
| 4. 권한상승 | 읽기 Agent가 쓰기·발송 도구 획득 | 도구별 capability token, 기본 읽기 전용, 런타임 권한 고정 | 예상 밖 도구 호출·권한거부 로그 | 토큰 폐기·세션 종료 |
| 5. 도구실행 | 신청·삭제·메일·결제·파일변경 | 매개변수 스키마, 대상 허용목록, dry-run, 고위험 HITL | 호출 전후 diff·대상·요청자 로그 | 취소 API·버전복원·보상 트랜잭션 |
| 6. 횡적이동 | 한 MCP의 악성 설명이 다른 도구 선택을 조작 | MCP 서버별 격리, 도구 설명 검수·서명, 비밀 분리 | 새 도구·설명·버전 변경 감시 | 서버 비활성화·자격증명 회전 |
| 7. 유출·영향 | 개인정보 전송, 잘못된 신청·대량행위 | 데이터 최소화·마스킹·속도제한·승인한도 | DLP·대량호출·외부전송 경보 | 전송 중단·키 회전·사용자 통지·사고대응 |

## 5. 고위험 행위 승인기준

반드시 실행 전 인간에게 `대상·행위·전송정보·예상결과·되돌리기`를 보여주고 명시 승인을 받는다.

- 신청·예약·취소·제출
- 외부 메시지·메일 발송
- 개인정보·건강·소득·가족정보 전송
- 파일·DB·메모리 영구 변경
- 유료 API·결제·계약
- 새로운 MCP·플러그인·Skill 설치

조회라도 주민번호·건강·소득정보가 포함되면 민감행위로 본다.

## 6. URL·출처·데이터 통제

- 공식 도메인과 구체 상세페이지 URL을 서비스별로 저장
- `source_url`, `retrieved_at`, `as_of`, 문서 해시, 추출방식, 검토상태 기록
- 리다이렉트 최종 도메인과 HTTPS 확인
- 외부 링크 응답을 DOM에 넣을 때 escaping·sanitize 적용
- 공식근거와 사용자별 판정을 분리하며 모델이 자격을 확정하지 않음
- 정책 수치 변경 시 기존 버전을 보존하고 diff·검토자 기록

## 7. 로그·롤백 최소요건

- 요청자·세션·Agent ID
- 사용 모델·프롬프트 정책 버전
- 읽은 출처 URL·해시
- 선택한 도구·권한·매개변수(민감값 마스킹)
- 인간 승인자·승인시각
- 실행 전후 상태와 결과
- 오류·차단 사유
- rollback ID 또는 원복절차

보안로그에는 원문 개인정보를 저장하지 않고 참조 ID·마스킹값을 쓴다.
