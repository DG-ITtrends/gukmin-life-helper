# 국민생활도우미

국민에게 생긴 사건을 출발점으로 공공서비스, 기관, 준비정보, 절차, 반복 제출 데이터와 AI·제도개선 후보를 연결한 정적 웹 MVP입니다.

대표 3개 사건(출생·실직·부모님 돌봄)은 정부24, 고용24, 고용노동부, 국민건강보험공단, 국가법령정보센터 등 공식 문서를 추가 대조해 신청시기·핵심 절차·법적 근거·주의사항을 심화 보강했습니다.

## 현재 범위

- 10개 생활사건: 출생, 이사, 실직, 폐업, 장애, 노인돌봄, 가족 사망, 전세피해, 재난, 해외 긴급상황
- 사건·서비스·기관 통합검색
- 분야 필터
- 사건별 서비스·국민여정·준비정보·공식 출처
- 최대 3개 사건 비교
- 반복 제출정보·병목·AI 개선기회 진단
- 반응형 모바일 UI

> 비공식 프로토타입입니다. 공식 출처 링크를 제공하지만 개별 요건의 법률적·행정적 타당성은 전문가 검수 전입니다.

## 실행

```bash
npm install
npm run dev
```

브라우저에서 표시된 로컬 주소를 엽니다.

## 검증

```bash
npm test
npm run build
```

정적 배포 산출물은 `dist/`에 생성됩니다.

## 데이터 구조

원본 데이터는 `src/data/journeys.js`에 있습니다. 사건별 필수 항목:

- 기본정보: slug, title, category, summary, asOf
- services: 서비스, 기관, 이용자 행동
- steps: 국민 관점의 순차 여정
- documents: 준비정보·서류
- reuseCandidates: 정부 보유정보 재사용 후보
- friction: 서비스 단절·병목
- aiOpportunities: AI 및 제도개선 후보
- sources: 공식 출처

## 공개 전 필요한 것

1. 10개 사건별 소관기관 또는 분야 전문가 내용 검수
2. 서비스별 정확한 신청 URL·요건·기한 보강
3. 프로젝트 명칭·로고·공개 라이선스 결정
4. GitHub Pages 또는 별도 도메인 선택

API 키, 서버, 데이터베이스는 현재 MVP 실행에 필요하지 않습니다.
